## 一、原始论文方案：DrseNet 数据泄露版本（2025年5-7月）

### 1.1 方案设计

原始论文围绕 BESD 数据集（儿童双语情绪语音数据集），构建了基于 DrseCNN 的儿童语音情绪识别模型。任务为 6 类情绪分类（ANGER、DISGUST、FEAR、HAPPY、NEUTRAL、SAD），评估指标为 Accuracy 和 Macro F1。

**特征工程**：对每个 WAV 文件提取 94 维（注：实际代码为 162 维）手工声学特征——ZCR(1)、Chroma(12)、MFCC(20)、RMS(1)、Mel(128)——然后对所有帧取 mean pooling，得到一个 162 维的固定长度向量。论文声称 94 维，但 librosa 默认参数 `n_mfcc=20, n_mels=128` 实际产生 162 维，多处注释存在错误。

**数据增强**：每个原始 WAV 做 3x 处理——（1）原始特征、（2）添加随机噪声（noise_amp=0.035）特征、（3）时间拉伸 + 音高偏移特征。处理后共约 12,537 条样本。论文将此称为"动态数据增强策略"，声称增加了数据多样性和模型鲁棒性。

**模型架构**：DrseCNN（Deep Residual Squeeze-Excitation CNN），核心设计为：
- 多阶段卷积结构，分层调整卷积核尺寸和通道数
- ResSE Block：残差连接（ResNet） + SE 通道注意力模块
- 自适应池化 → 全连接分类器
- 参数量约 45.5M（完整版）

**消融实验**：设计了四组对比——BaseCNN（纯卷积）、ResCNN（+残差）、SECNN（+SE注意力）、DrseCNN（完整版）。另与参考文献 [7] CNN、[15] CNN-BiLSTM、[17] Transformer 进行对比。

### 1.2 论文中的实验结果

| 模型 | Accuracy | F1-Score | 参数量 | 收敛速度 |
|------|----------|----------|--------|---------|
| **DrseCNN (本文)** | **86.82%** | 86.00% | 11.37M | 35 epochs |
| CNN-BiLSTM [15] | 72.26% | 71.70% | 3.02M | 62 epochs |
| CNN [7] | 72.14% | 71.76% | 1.46M | 40 epochs |
| Transformer [17] | 69.48% | 69.25% | 4.80M | 50 epochs |

消融实验中 DrseCNN 最佳验证准确率达到 **85.99%**（Acc=0.8599, F1=0.8597）。

### 1.3 论文中未说明的关键问题（共 9 项，由一次面试触发发现）

**发现问题的起因**：在一次求职面试中，向面试官（公司 CTO）介绍本项目的"动态数据增强策略"时，CTO 当场提出质疑——"先增强再随机划分"的顺序可能导致同一条原始语音的增强版本同时出现在训练集和测试集中。面试后我立即核查代码，不仅确认了数据泄露的存在，还顺藤摸瓜发现了以下全部问题。

**问题 1 — 数据泄露（最严重，面试直接触发）**：论文第 V 章声称按说话人（70%/30%）划分训练/测试集，确保"训练和测试数据来自不同说话人"。经代码审查，实际实现为：

```
① englishdata.py: random.shuffle(all_data) → 70/30 切分生成 data_all.npy
② train.py: random_split / random.shuffle 在整个数据集上打乱
③ 没有任何代码从文件名提取说话人 ID 或按说话人分组
④ 原始 4,179 条 × 3 增强版本 → 12,537 条 → 随机 70/30
   → 同一原始语音的 3 个增强版本被分到不同集合的概率约 63%
```

**量化影响**：约 63% 的原始语音，其增强版本同时出现在训练集和测试集中。模型学到的是"同一句话的三种变体听起来很像"而非"儿童语音的情感特征泛化"。

**问题 2 — 数据增强顺序错误**：流程为"原始数据 → 3x 增强 → random split"，正确流程应为"原始数据 → 按说话人划分 → 仅对训练集增强"。测试集中包含增强数据，且与训练集中同一原始语音的增强版本重叠。

**问题 3 — 时间维度完全丢失（架构错配）**：`extract_features()` 对每一帧特征在时间轴上取 mean：
```
WAV (2.5s @ 16kHz = 40,000 samples)
  → librosa 帧级特征 (T 帧 × 162-dim)
  → np.mean(所有帧, axis=0)      ← 时间维度被抹平
  → (162,) 固定长度向量
```

DrseCNN 的 Conv1d kernel 在 `[ZCR(1), Chroma(12), MFCC(20), RMS(1), Mel(128)]` 的**特征拼接维度**上滑动，而非时间轴。ResSEBlock 和 MaxPool1d 设计本意是捕捉时序结构，但输入已完全丢弃了时序信息。对比模型同样受影响——Transformer 输入 (B, 1, 162) → T=1 → self-attention 退化为恒等映射，CRNN 的 LSTM 把下采样后的 ~40 个位置当时间步，而位置 i 到 i+1 跨越的是不同特征组的边界。

**问题 4 — 数据集选择不当**：代码仅使用 BESD 的 ENGLISH 子集，论文未说明这一限制。MY = ENGLISH + TELUGU 才是完整数据集（4,179 WAV，237 名说话人）。

**问题 5 — 说话人 ID 解析缺陷**：文件名存在多种格式变体（`1.EF-12` vs `1.EF_12`、`3EF_9` 缺失点号），原解析逻辑未处理特殊格式，导致同一说话人可能被拆分为多个 ID。

**问题 6 — 特征维度标注错误**：多处注释和论文表 1 标注为 94 维，实际 162 维。

**问题 7 — 网络结构维度硬编码**：`CNNModel` 全连接层输入维度硬编码为 4608、`CNNBiLSTMModel` 硬编码 `input_dim=94`。

**问题 8 — ONNX 兼容性**：多处使用 `AdaptiveAvgPool1d`，该算子不被 ONNX 运行时支持。

**问题 9 — 论文数据不一致**：论文声称 4,320 条语音，实际 4,179 条；表 1 中 F0 均值 298.11 Hz 实际为 291.10 Hz 等。

### 1.4 小结

第一阶段的实验结果（86.82%）完全无法采信。根本原因是 **speaker 重叠导致的数据泄露 + 所有模型架构与输入错配**。论文的"模型创新"叙事建立在一个不可靠的实验基础上。**这次面试经历让我深刻认识到：实验结果不仅要看数值，更要看实验流程是否经得起推敲。**

## 二、修复数据泄露后的 DrseCNN 版本（2026年4月25-27日）

### 2.1 修复措施

面试后立即对项目进行了系统性修复：

| 修复项 | 具体操作 |
|--------|---------|
| **数据泄露** | 重写 `preprocess.py`：从文件名提取说话人 ID → `normalize_speaker_id()` 修正格式变体 → profile-based 分层划分（按说话人的情感类别组合分组，组内 70/30）→ 165 训练/72 测试，**零说话人重叠** |
| **数据集切换** |  MY（ENGLISH + TELUGU），4,179 WAV, 237 说话人 |
| **增强顺序修正** | 训练集和测试集均做 3x 处理——应视为"特征提取策略"而非"数据增强"（论文称"动态数据增强"从而被认定为泄漏） |
| **维度注释修正** | 94 → 162，全文件逐行修正 |
| **CNNModel ONNX 兼容** | `AdaptiveAvgPool1d` → `MaxPool1d` + 动态维度推导（dummy forward 计算 flat_dim） |
| **旧 checkpoint 归档** | 全部移至 `checkpoints/legacy/`，标注为基于泄露数据无效 |
| **评估脚本去重** | 移除 `evaluate_model.py` 中重复定义的 `NpyDataset`/`collate_fn`，统一从 `src.data.dataset` 导入 |

### 2.2 修复后的实验结果

在无数据泄露的严格 speaker-independent 划分下，使用原 162-dim mean-pooled 特征，重新训练各模型：

| 模型 | 修复前（含泄露） | 修复后（无泄露） |
|------|:---:|:---:|
| DrseCNN | 85.99% | **~35%** |
| CNN | 72.14% | ~30% |
| BiLSTM | 72.26% | ~30% |
| Transformer | 69.48% | ~28% |

准确率从 86% 直接跌落到 **30-40%**，各模型间的"性能梯度"几乎消失。六个类别的随机猜测基线约为 16.7%，30-40% 仅仅是略高于随机水平。

### 2.3 根因分析：为什么 162-dim mean-pooled 只有 30-40%？

2025-04-27 讨论中确认的核心问题：

**所有模型都在没有时间维度的数据上做"时序建模"**。162 维均值向量本身携带的情绪信息上限就是 30-40%——无论用什么 fancy 的网络架构（ResNet+SE、Transformer、CRNN）都无法突破这个信息瓶颈。

具体来说：
- **DrseCNN Conv1d**：kernel 在 `[ZCR, Chroma, MFCC, RMS, Mel]` 的特征拼接维度上滑动 → 不是时间卷积
- **Transformer**：(B, 1, 162) → T=1 → self-attention 完全无意义
- **CRNN**：CNN 下采样后 LSTM 的"时间步"其实是不同特征组之间的边界
- **BiLSTM**：需要 T>1 的时序输入，却被喂入 T=1 向量

第一阶段 86.82% 的高准确率完全来自 **speaker 重叠**——模型学到了"这个说话人像训练集中的谁"，而非真正的 emotion recognition。

### 2.4 这个阶段的关键结论

1. **数据泄露是 SILENT KILLER**：86% → 35%，幅度巨大，且原方案没有任何机制能检测到
2. **特征表示是根本瓶颈**：162-dim mean-pooled 的信息上限 ≈ 35%，不是模型不够好，是特征从根本上错了
3. **论文中 6 个基准模型的"性能梯度"本质上是数据泄露程度的不同响应**，而非真正的模型能力差异
4. **Speaker-independent 划分是儿童 SER 的底线要求**：不满足此条件的结果全部无效

## 三、当前新方案：分布驱动三模块框架（2026年4月27-28日）

### 3.1 方向的确立：又一次面试带来的启发

在确认 162-dim mean-pooled 是信息瓶颈后，我的第一反应是尝试自监督学习（SSL）。当时选了 wav2vec2，但调参后效果不佳，准确率比 35% 好不了多少。于是我初步判断"自监督学习跟这个任务不太匹配"，准备放弃这条路。

**转折发生在入职现在这家公司前的面试。** 跟 CTO 聊起项目经历时，CTO 听完我的描述后指出：

> "wav2vec2 效果不好，不代表自监督学习这条路不对。wav2vec2 是 2020 年的模型，预训练数据是成人朗读的 LibriSpeech，跟儿童语音本身就有域差。而且 SSL 模型调参空间大——学习率、冻结层数、是否 fine-tune——你调参没到位也很正常。你 162-dim mean-pooled 的问题是丢了时间维，而 SSL 模型输出的正是帧级特征，天然保留了时间结构。你需要的是换个更适合的模型，不是否定整个方向。"

CTO 从原理层面分析了 WavLM（语音下游任务全面优于 wav2vec2）和 emotion2vec（专门为情绪识别设计）的特点，建议我重新走 SSL 方向。

这次谈话让我意识到：之前放弃 SSL 不是因为方向错了，而是模型选型 + 调参的问题。

**进一步思考**：直接拿成人预训练模型用在儿童语音上还不够——儿童语音的 F0 分布、语速模式、韵律特征跟成人有系统性差异。由此形成了"分布驱动三模块"的整体思路：

```
SSL 帧级特征（地基） + 三个面向儿童分布特点的适配模块
```

核心叙事从"提出新网络模型"转变为"儿童语音的统计分布特性驱动 SER 流程设计"：

```
儿童语音 ≠ 成人语音（定量统计分布分析）
         ↓
分布偏移在 SER 三个环节的具体表现
         ↓
每个环节对应一个技术模块（由分布分析"逼出来"的设计）
         ↓
分类器用 DrseNet（不是核心创新，只是工具性组件）
```

**关键决策**：
1. **特征升级**：162-dim mean-pooled → WavLM/emotion2vec 帧级 768-dim (T×768)，**保留时间轴**（200 帧 @ 50Hz）
2. **架构修正**：DrseNet 从"核心贡献"降级为"分类器"，一句话带过
3. **创新重点**：三个由儿童语音分布特性驱动的技术模块

### 3.2 Phase 1：验证 SSL 基线的有效性（4月27日）

**目标**：验证 CTO 的判断——SSL 帧级特征是否能解决 30-40% 的瓶颈。

**做法**：
- 预提取 BESD MY 全部 4,179 条 WAV 的 WavLM/emotion2vec 帧级特征
- 统一 pad/truncate 到 200 帧（4s @ 50Hz），保存为 `(N, 200, 768)` float32 数组
- Frozen backbone + DrseCNN 分类器 linear probe

**结果**：

| 模型 | 最佳 Val Acc | 备注 |
|------|-------------|------|
| 旧基线（162-dim mean-pooled） | ~35% | 不可用 |
| emotion2vec Base | ~66% | 震荡较大（50-66%） |
| **WavLM Base** | **80.66%** | 稳定上升，方差小 |

**结论**：CTO 的判断完全正确——WavLM 帧级 768-dim + 保留时间维 → 80.66%，远超 40% 阈值。选对模型确实关键：wav2vec2 跟 WavLM 在儿童 SER 上的差距远比预期的要大。确认方向正确。论文以 WavLM 为主模型，emotion2vec 留作消融对比。

### 3.3 Phase 2：三个模块逐个实现

#### 模块 1：分布校准适配器（AcousticCalibrationAdapter）

**一句话概括**：给 WavLM 这只"成人耳朵"配一副"儿童矫正眼镜"。

**动机**：WavLM/emotion2vec 在成人语音数据上预训练，统计分布分析发现儿童与成人在 SSL 隐空间中确实存在可测量的分布偏移（Frechet Distance = 0.37，维度均值的 scale 平均偏离 8%）。

**计算**：从 BESD MY（儿童）和 IEMOCAP（成人，9,903 条）分别提取 WavLM 帧级特征，计算 per-dimension mean/std，得到 `scale` 和 `bias` 各 768 维。

**设计——Adapter 到底是什么？**

Adapter 是一个插在 frozen WavLM 之后的轻量校准层，结构分两段：

- **第一段（线性校准）**：`output = x × scale + bias`。768 个 scale + 768 个 bias，共 1,536 参数。对每个特征维度做全局矫正——哪个维度被压低了就放大，压高了就压下去。
- **第二段（逐帧微调）**：一个小 gate 网络（768→192→768，ReLU + Tanh），约 30 万参数。看每一帧的内容，输出微调信号，以 0.1 倍幅度叠加到第一段结果上。元音帧和辅音帧的偏移方向不同，愤怒和悲伤的偏移也不同——第二段负责处理这些帧级差异。
- 总参数约 30 万，仅占 WavLM 94M 的 0.3%。

**初始化方式**：
- 统计先验初始化（用 child→adult 的 scale/bias 补偿，相当于"算出眼镜度数"）
- 随机初始化（scale 全 1、bias 全 0，让模型自己学，相当于"让耳朵自己适应"）

**实验结果**：

| 实验 | 配置 | Val Acc | vs 基线 |
|------|------|---------|---------|
| A1 | 无 Adapter（linear probe） | 80.66% | — |
| A2 | Adapter + 统计先验初始化 | 81.45% | +0.79pp |
| **A2b** | **Adapter + 随机初始化** | **85.40%** | **+4.74pp** |

**反直觉发现**：统计先验初始化不如随机初始化。原因：全局 per-dimension 补偿抹平了不同帧（元音 vs 辅音 vs 静音）、不同说话人、不同情绪的差异性偏移方向。好比所有场景用同一副固定度数的眼镜——远景清晰了近景就模糊。随机初始化 + 梯度下降能学到"对分类任务有利"的校准方向，而非死板地对齐分布。这个发现对论文反而是正面证据——证明分布偏移不是简单的"平移+缩放"，而是结构性的、非线性的，需要可学习的非线性校准，这正是 Adapter 存在的价值。

#### 模块 2：时序重要性池化（TemporalImportancePooling）

**一句话概括**：用 F0 和能量告诉模型"应该重点听哪几帧"，而非对所有帧一视同仁地取平均。

**动机**：儿童语音 F0 波动范围极宽（p5=75.3Hz, p95=2285.7Hz, mean=495.9Hz, std=651.1Hz）、语速波动大，帧级情感信息密度极不均匀。原有 mean pooling 对所有帧一视同仁取平均，关键情绪瞬间被稀释。

**设计——不是 attention，是韵律引导的帧级加权：**

具体分三步：
1. **提取韵律特征**：librosa 对每帧提取 F0（基频）+ Energy（RMS 能量）→ (B, T, 2)
2. **韵律引导的打分**：韵律 (B,T,2) 投影到 64 维 + SSL (B,T,768) 拼接为 (B,T,832)，过小网络 (832→128→1) 为每帧打一个分数
3. **加权求和**：Softmax 归一化 → 加权求和 → (B, 768)

**与标准 self-attention 的本质区别：**

| | 标准 self-attention | 韵律引导池化 |
|---|---|---|
| 权重来源 | 纯数据驱动：Q、K、V 全来自 SSL 特征 | 声学知识注入：F0/能量作为外部先验 |
| 可解释性 | "模型觉得这帧重要"（不知道为什么） | "F0 突变帧获得高权重"（对应愤怒/开心语调起伏） |
| 物理直觉 | 无 | 有：音高剧烈变化→可能含情绪转折→高权重 |

**为什么儿童语音特别需要？** 同一句话里，儿童音高可能从 75Hz 飙到 2286Hz——帧间差异极大。愤怒时 F0 陡然升高，那几帧至关重要；悲伤时能量持续低落。Mean pooling 把 200 帧压成一个平均数，关键信息被稀释。

**实验结果**：Adapter(随机) + Prosody Pooling → 86.42% Val Acc（+1.02pp over Adapter-only），消融对比中韵律引导 > 纯 self-attention

#### 模块 3：负面实验 —— 儿童语音情感对波形增强的脆弱性分析

**⚠️ 定位澄清：这个模块不是提出一种"新的增强方法"。它是一个负面实验（Negative Experiment），目的是系统性证明"增强这件事在儿童语音上需要重新审视"。C1（无增强）= 86.42% 是最高准确率，所有增强组全部掉点——这不是 bug，这正是我们要证明的论点。**

**动机**：成人语音上有效的增强参数（如 pitch shift ±6 semitones, time stretch 0.7-1.3）应用到儿童语音上可能使其脱离真实分布，破坏情感特征。成年人 F0 ~120Hz，儿童 F0 均值 496Hz——同样的 ±6st，成人移几十 Hz，儿童移几百 Hz，失真程度完全不同。

**FD（Fréchet Distance）是什么？**：FD 衡量增强后样本的 WavLM 隐空间分布与原始样本分布之间的距离。FD=0 表示完全一致，FD 越大表示增强把样本推得离原始分布越远。可以理解为"分布偏移的定量标尺"。

**实验设计**：四组使用完全相同的模型（WavLM + Adapter + Prosody Pooling），唯一变量是训练数据的增强方式。离线预增强 + WavLM 特征提取后训练。

**负面实验结果**：

| 实验 | 增强参数 | FD (偏移量) | Val Acc | vs C1 |
|------|---------|------------|---------|-------|
| **C1** (无增强) | — | 0 | **86.42%** | 基线 |
| C3 (儿童约束) | ±3st, 0.85-1.15 | 8.71 | 66.85% | **-19.57pp** |
| C2 (成人参数) | ±6st, 0.7-1.3 | 9.87 | 59.51% | -26.91pp |
| C4 (极端参数) | ±12st, 0.5-1.5 | 11.99 | 43.25% | -43.17pp |

**关键发现**：

1. **C1 无增强 = 最高准确率**。这证明了"不做增强"就是儿童语音 SER 的最佳策略。儿童语音中的情感信息对波形操作极其脆弱。
2. **FD 与准确率呈现严格单调负相关**——FD↑ → Acc↓，四组无一例外。证明分布偏移可定量预测分类退化。
3. **即使"儿童约束"参数也 -19.57pp**。说明仅靠"缩小参数范围"无法解决问题——需要从根本上重新思考"在儿童语音上该不该做增强"。

**为什么无增强效果最好还要做这个实验？**

因为这个实验的目的不是找到更好的增强方法，而是**证明增强这件事在儿童语音上就需要重新审视**。它为整个框架的"分布驱动"叙事提供了最强有力的实验基础——正因为增强如此容易破坏儿童语音情感特征，所以每个环节都需要从分布出发重新设计。

**论文中的定位**：属于实验章节（Experiments），不叫"分布约束增强方法"，而叫"负面实验：增强对儿童语音情感的破坏性分析"。大多数论文只报告"加了某模块提升多少"，我们系统地展示"如果不加约束，性能会怎样逐级退化"——这种负面实验范式在 SER 领域非常罕见。

对论文的价值：这张 FD vs Accuracy 图是全文最有说服力的图表，证明"分布偏移可定量预测分类退化"，为三模块框架的核心叙事提供了实验基础。

### 3.4 Phase 3：整合实验与数据泄露再修复（4月28日）

**第二次数据泄露修复**：

在 Phase 2 完成后审查发现两个新增泄露风险，且这两个风险比第一次更隐蔽：

**风险 1 — adapter_init 统计先验泄露（为什么 70/30 的 adapter_init 偷看了验证集？）**

`compute_adapter_init.py` 计算 adapter 的统计先验（per-dim mean/std of child vs adult SSL features）时，使用的是**全部 237 个 speaker** 的数据，包括 test 中的 72 人。这意味着：
- adapter 的 scale/bias 初始参数（各 768 维）中编码了测试集说话人的分布信息
- 虽然不直接包含标签，但模型在训练开始前就"知道"测试集的统计特征（哪些维度的方差大、均值偏多少）
- 相当于老师根据"全班所有学生的平均水平"设计了一张矫正表——但如果"全班学生"包括了期末考试的考生，这张表就提前编码了他们的信息

**风险 2 — 测试集当验证集做 early stopping（间接泄露）**

70/30 划分只有 train 和 test 两个集合，没有独立的验证集。early stopping 只能在 test 上做——每次选择"test 上表现最好的 epoch"保存模型。test 间接参与了模型选择，相当于考试前老师把试卷给你看了几次——虽然不是直接把答案告诉你，但你根据"哪个版本的答案得分最高"来调整策略。

**修复方案：60/20/20 三路划分**

| 风险 | 修复 |
|------|------|
| adapter_init 使用了全部 speaker（含 test） | `compute_adapter_init.py` 改为 `split_speakers_3way()` 仅用 train 的 60% 说话人计算 |
| test set 当 val 做 early stopping | `split_speakers_3way()` 60/20/20 → val 做 early-stop，test 严格 hold-out |

新增 `split_speakers_3way()` 函数，profile-stratified 三路划分，三层断言保证互斥。

**论文中如何写 6/2/2 划分？**

方案 A（推荐）：直接写"将数据集按说话人划分为训练集(60%)、验证集(20%)和测试集(20%)"——这是深度学习论文的标准做法，train/val/test 三路划分非常普遍。

方案 B（传统）：写"按 8:2 划分训练集和测试集，并从训练集中划出 15% 的说话人作为验证集用于早停"——实质上等价于三集合划分，但形式上保持了 train/test 二路划分的传统表述，适用于对三路划分有疑虑的审稿环境。

两种方案的实质完全相同，选择哪种取决于目标期刊/会议的惯例。

**云端全消融实验**（RTX 4090D 24GB，3-way split，3 seeds × 5 configs = 15 runs）：

| 实验 | Adapter | Pooling | Backbone | Val (mean±std) | Test (mean±std) |
|------|---------|---------|----------|---------------|-----------------|
| A1 基线 | ✗ | mean | frozen | 85.68% ± 0.22% | **78.34% ± 1.07%** |
| A2 统计先验 | ✓ stat | mean | frozen | 86.32% ± 0.26% | 79.25% ± 0.05% |
| A2b 随机初始化 | ✓ rand | mean | frozen | 86.40% ± 0.55% | 79.51% ± 0.74% |
| **B3 Adapter+Prosody** | ✓ rand | prosody | frozen | **88.52% ± 0.49%** | **81.24% ± 1.45%** |
| **A3 Full Fine-tune** | ✓ rand | prosody | unfrozen | 88.52% ± 0.32% | **81.96% ± 1.31%** |

**模块贡献分解**（Test Acc 视角）：

| 模块 | Test Acc 增量 | 备注 |
|------|-------------|------|
| + WavLM (vs 旧162-dim) | ~43pp | Phase 1 已验证 |
| + Adapter (random init) | +1.17pp | A1→A2b |
| + Prosody Pooling | +1.73pp | A2b→B3 |
| + Full Fine-tune | +0.72pp | B3→A3 |
| **从旧基线累加** | **+46.96pp** | ~35% → 81.96% |

**关键发现**：
1. Adapter 统计先验(79.25%)≈随机(79.51%)，结构本身是收益来源
2. 韵律引导 > 纯 mean pooling（+1.73pp），且有可解释性优势
3. Full fine-tune 边际收益仅 +0.72pp，说明轻量模块（~900K 参数）已捕获大部分收益
