## 一、核心发现：当前项目 30-40% 准确率的根因

所有模型在 **没有时间维度的数据上做时序建模**。

```
WAV (2.5s) → librosa 帧级特征 → np.mean(所有帧) → (162,) → Conv1d
```

- Conv1d kernel 在 `[ZCR(1), Chroma(12), MFCC(20), RMS(1), Mel(128)]` 的拼接特征上滑动，而非时间轴
- Transformer 输入 (B, 1, 162) → T=1 → attention 无意义
- CRNN 的 LSTM 把下采样后的 ~40 个"位置"当时间步 —— 位置 i 到 i+1 是不同特征组的边界

原始论文 86.82% 完全来自 speaker 重叠导致的数据泄露。去掉泄露后，30-40% 是 162 维均值向量本身能携带的情绪信息上限。

## 三、"分布分析驱动"框架（最终版本 v5）

### 核心叙事

```
儿童语音 ≠ 成人语音（定量统计分布分析）
         ↓
分布偏移在 SER 两个关键环节的具体表现
         ↓
两个技术模块（被"逼出来的"设计）
         ↓
分类器用 DrseNet（不是核心创新，只是自然汇聚的结果）
         ↓
增强实验作为负面验证（证明成人增强经验不可迁移）
```

### 两个技术模块（方法章节）

#### 模块 1：声学校准适配器（Acoustic Calibration Adapter）→ 降级为消融讨论

**一句话**：解决"每一帧的特征对不对"。

**问题**：WavLM 在成人数据上预训练，儿童语音在其隐空间中存在系统性分布偏移（FD=0.37）。帧级特征有域偏移。

**方法**：在 frozen WavLM 输出后加一个轻量校准层（scale + bias + gate），对每一帧的 768 维特征做自适应校准。

**实验结论（v5 6:2:2）**：Adapter 对测试准确率的贡献仅 ~0.5pp（A3→B3: 80.85%→81.47%），单独使用几乎无增益（A1→A2b: 78.61%→78.81%）。**论文中建议将 Adapter 从方法章节移除，作为消融讨论的一部分。**

#### 模块 2：韵律引导的时序重要性池化（Prosody-guided Temporal Importance Pooling）→ 核心方法

**一句话**：解决"哪些帧更重要"。

**关键实验（v5 6:2:2）**：A1 (mean pool)→A3 (prosody pool): +2.24pp test (78.61%→80.85%)。**这是全框架的主驱动力。**

### 模块 1 和模块 2 的关系（简化后）

```
WavLM  →  Prosody Pooling  →  DrseNet
帧级特征   选择/加权重要时间帧      最终分类
(200×768)  (768)                  (6 classes)
```

Adapter 作为可选附加模块，不在核心方法路径中。论文叙事从"两个模块"简化为"一个核心模块 + 分布偏移分析"。

### 增强敏感性分析（实验章节，不是方法章节）

**定位**：模块 3 不作为"提升性能的方法模块"，而是负面实验（Negative Experiment）。

**问题**：成人 SER 中常用的数据增强（pitch shift, time stretch）应用到儿童语音上，会引入分布偏移并破坏情感相关特征。

**实验设计**：
- C1 (无增强): baseline
- C2 (成人参数): pitch ±6, stretch 0.7-1.3
- C3 (儿童约束): pitch ±3, stretch 0.85-1.15
- C4 (极端参数): pitch ±12, stretch 0.5-1.5

**核心发现**：C1 无增强 86.42% 最高，所有增强组全部下降。FD 与准确率严格负相关（FD越大→准确率越低），证明分布偏移可定量预测分类退化。

**论文中的论证价值**：
- 实验目的不是为了"提出新增强方法"
- 而是证明成人 SER 中标配的增强不能直接迁移到儿童 SER
- FD ∝ 1/Accuracy 的单调关系为"分布偏移→分类退化"提供了定量证据
- 最终推荐模型明确写"不使用波形增强"

### 数据划分协议（最终版本 v5）

外部 8:2 + 内部 val：
1. 外部 speaker-independent 8:2：80% speaker → trainval, 20% speaker → test
2. 内部从 trainval 中划分 val（~15% → 最终约 train 68%, val 12%, test 20%）
3. adapter_init 只用 train speaker
4. val 仅用于 early stopping 和模型选择
5. test 严格 hold-out，只在最终评估时使用一次
