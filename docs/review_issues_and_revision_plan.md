## 第二部分：为什么操作严谨还会出这些问题

### 根因 1：写作速度 > 质量检查

Phase 6.5 论文撰写在一个小时内完成，没有经过冷却期重新审视。每个实验做完就立刻写进论文，没有回头检查各章节之间的一致性。

**表现为**：
- §1/§4.3/§6 三处表述矛盾
- FD_lang 估计值写成了"高达约 137"像是精确值
- "两个数量级"的计算错误（137/12=11.4 倍）

### 根因 2：实验与叙事脱节

实验设计时关注的是"跑出数据"，写作时才开始"编故事"。导致叙事方向与数据不完全匹配。

**表现为**：
- 跑完 CREMA-D 得到 +0.75，立刻塞进"年龄×风格"框架，但没有验证这个框架是否真的成立
- FD 框架是先有概念后有数据，但数据只能支撑"关联"而非"预测"
- 注意力分析跑出来 entropy 范围极窄（5.17-5.20），差异可能来自噪声

### 根因 3：缺乏统计思维

所有实验只报告了均值和标准差，但没有做统计检验来验证差值的显著性。

**表现为**：
- A1 vs A2b 的 0.2pp 差值是否有统计意义？
- IEMOCAP Δ=−2.12 是否显著？
- 6 类熵-准确率相关性是否显著？

### 根因 4：跨阶段协议不一致

Phase 4（跨语言实验）用了旧的 60/20/20 split，到 Phase 6.2 切换为 6:2:2，但跨语言数据未重新提取。

**表现为**：§4.6 的 X1/X2 数据和 §4.2 的主实验数据来自不同 split 协议，无法直接比较。

### 根因 5：没有冷静期审查

论文写完立刻被视为"完成"，没有留出 24 小时的冷却期重新阅读。所有作者都经历过——写完觉得自己写得很好，第二天再看就发现一堆问题。

## 第三部分：修改方案

### 第一步：数据补全（1-2 天）

| 补项 | 方法 |
|------|------|
| A1 vs A2b 统计检验 | paired t-test 或 bootstrap CI |
| IEMOCAP A3 Δ 置信区间 | bootstrap 95% CI |
| 注意力熵-准确率 Pearson r | 计算 r 和 p 值 |
| 三数据集 F0/能量统计 | 从 prosody .npy 计算均值/方差 |
| FD_lang 真值 | 提取 ENGLISH/TELUGU 子集特征计算 |
| 6.2:2 协议下的跨语言数据 | 用新 split 重新提取（或标注协议差异） |

### 第二步：逻辑一致性修正（0.5 天）

| 修改点 | 做法 |
|--------|------|
| §1/§4.3/§6 统一表述 | 全部改为"与年龄和情绪产生方式相关"，取消"调节""交叉效应"等词 |
| §4.2 76% 争议 | 改为"Rao 未明确描述 split 方法；我们在严格协议下得到 50.2%"。不强推结论 |

| §1 引言结构 | 删掉"本文核心发现包括"，挪入 §6 |

逐句将"证实/揭示/取决于/迫使/所有"替换为"表明/提示/与...相关/引导/在本实验条件下"。

## 第四部分：以后如何避免

### 流程改进

1. **冷却期制度**：论文写完至少放 24 小时再自审
2. **逻辑一致性检查表**：提交前逐条核对（Abst/Intro/Method/Exp/Conclusion 的叙事是否一致）
3. **统计显著性报告**：凡是差值均报告 CI 或 p 值
4. **协议一致性审计**：确保所有实验使用同一 split 协议
5. **审稿人模拟**：写完初稿后，强迫自己逐段扮演审稿人写质疑

### 写作原则

1. **先有数据后有叙事**：不要为了"故事好看"而扭曲数据的方向
2. **少用因果词**：除非有对照实验支撑，否则用"相关""提示""观察"而非"证实""揭示""取决于"
3. **数字必须可追溯**：每个引用的数值都标注来源（自己算的 vs 文献引的 vs 估计的）
4. **不强推单薄结论**：三个数据点就说"提示了某种关联"，六个数据点加上统计检验才能说"呈显著相关"
