# 已知问题
## Known Issues

## 概述

本文档记录项目实现中的已知问题、限制和待改进之处。这些问题不影响项目核心价值的展示，但需要在技术交流和实际应用时注意。

## 项目历史背景

1. **多版本文件管理**: 项目文件历史上存在多个版本 (如 `code_屎山版/` 和 `源代码_交给团队版/`)，导致代码管理复杂。
2. **代码参考与协作**: 部分代码实现参考了开源项目，并在团队协作中开发，非完全独立实现。
3. **早期实验复现性**: 早期实验未固定随机种子，部分实验配置未完整记录。
4. **公开展示限制**: 当前GitHub公开展示版不包含大型检查点文件、原始数据集和预提取特征文件。
5. **直接运行障碍**: 当前版本需要用户自行准备数据集和生成特征文件，无法直接运行。

## 关键方法论问题（影响论文结论可信度）

### ⚠️ 1. 说话人无关划分未落实（数据泄露）

**问题**: 论文第V章声称按说话人（70%/30%）划分训练/测试集，确保"训练和测试数据来自不同说话人"。经审查，所有训练脚本实际使用的是对整个数据集做随机打乱划分（`random.shuffle` / `random_split`），未按说话人分组。

**证据**:
- `englishdata.py`（数据生成阶段）: `random.shuffle(all_data)` → 70/30 切分
- `train.py`（训练阶段）: `random_split` 或 `random.shuffle`
- 没有任何代码从文件名提取说话人ID或按说话人分组

**数据泄露量化**:
- 原始 4,180 条 × 3 增强版本 = 12,540 条
- 随机 70/30 划分下，同一原始语音的 3 个增强版本被分到不同集合的概率约 **63%**
- 即约 63% 的原始语音，其增强版本同时出现在训练集和测试集中

**影响**: 论文报告的 85.99% 准确率因数据泄露而**不可靠**。在严格的说话人无关划分下，真实准确率将显著低于该值。

**修正计划**: 见 ROADMAP.md 优先级1。

### ⚠️ 2. 数据增强顺序错误

**问题**: 当前流程为 原始数据 → 数据增强 → 划分训练/测试集。正确流程应为 原始数据 → 按说话人划分 → 仅对训练集增强。

**影响**: 测试集中包含增强后的数据，而这些数据与训练集中的增强版本可能来自同一原始语音，导致测试集并非完全"unseen"。

### 3. 两份 englishdata.py 不一致

**问题**: 项目中曾存在两份功能不同的数据预处理脚本:
| 版本 | 位置 | 增强状态 |
|------|------|----------|
| V1 | `code_屎山版/englishdata.py` | 增强被注释掉（同一个 feat 重复 3 次） |
| V2 | `src/data/dataset.py` | 增强激活（noise + stretch + pitch） |

**处理**: V2（`src/data/dataset.py`）为权威版本，V1 已归档。

## 网络结构问题

### 1. 输入维度不匹配（论文 vs 代码）

**问题**: 论文描述 94 维输入（ZCR + Chroma + MFCC + RMS + Mel），但实际 `collate_fn` 将 `(3, 94)` reshape 为 `(1, 282)`，网络实际接收 **282 维**。

**影响**: 他人按论文描述（94维）无法复现网络。

### 2. 输入尺寸固定

**问题**: 模型要求固定的输入时间长度和特征维度，当前实现假设输入为 `(B, 1, T, 94)`。

**影响**: 无法处理变长音频，特征维度改变时需要修改模型。

**改进方向**: 实现自适应池化或Transformer架构支持变长输入。

### 3. 过参数化风险

**问题**: DrseCNN参数量较大 (~45.5M)，而数据集有限（~12,540个样本），参数:样本比约 1:3600。

**影响**: 容易过拟合，需要强正则化；训练不稳定，需要精心调参。

**缓解措施**: 使用了Dropout（0.3→0.4→0.5）、权重衰减、数据增强等正则化技术。

### 4. 计算资源需求高

**问题**: DrseCNN训练需要较多GPU资源：GPU内存 10GB+，训练时间 8.5小时+，批量大小最大128。

**影响**: 资源受限环境下难以训练，实验迭代速度慢。

**解决方案**: 提供轻量版模型选项（CNN、小型Transformer）。

## 论文数据问题

### 1. 数据集规模不一致

- 论文声称: **4,320** 条语音记录
- 实际统计: **4,180** 个 .wav 文件（少 140 条）

### 2. 表1统计数据偏差

| 指标 | 论文值 | 实际计算值 | 偏差 |

| 基频 F0 (Hz) | 298.11±44.39 | 291.10±48.48 | -7.01 / +4.09 |
| 信噪比 (dB) | 13.61±7.72 | 15.63±5.97 | +2.02 / -1.75 |
| 静音长度 (s) | 0.46±0.14 | 0.40±0.27 | -0.06 / +0.13 |
| 音节速率 (syll/s) | 3.52±1.22 | 3.46±1.02 | -0.06 / -0.20 |

### 3. 引用错误

论文第II章引用 "Lesyk等人[9]"，但参考文献 [9] 实际为 Southwell 等人的研究，与 Lesyk 无关。

## 代码实现问题

### 1. 硬编码路径
**问题**: 多个文件中包含硬编码的本地路径
```python
# englishdata.py 中的示例
wav_dirs = r'C:\Users\59892\Desktop\数据集\BESD\BESD\MY'
```

**影响**:
- 可移植性差，在其他环境无法直接运行
- 需要手动修改路径才能使用

**临时解决方案**:
```python
# 修改为配置变量或命令行参数
import argparse
parser = argparse.ArgumentParser()
parser.add_argument('--data_path', type=str, default='./data')
args = parser.parse_args()
```

### 2. 随机种子未固定
**问题**: 早期实验脚本未设置随机种子
```python
# train.py 中缺失
# 应添加: torch.manual_seed(42)
```

**影响**:
- 实验结果不可完全复现
- 不同运行结果可能波动

**解决方案**: 已在`reproducibility-notes.md`中提供修复方法。

### 3. 版本冲突和重复代码
**问题**: 存在多个功能相似的脚本文件
```
code_屎山版/
├── train.py
├── train_2.py
├── train_3.py
├── train_4.py
└── train_5.py
```

**影响**:
- 代码维护困难
- 不确定哪个版本是最终版本

**处理方式**: GitHub展示版仅保留核心版本`src/training/train.py`。

## 数据处理问题

### 1. 数据增强参数固定
**问题**: 数据增强的幅度和方式是固定的
```python
def noise(data):
    noise_amp = 0.035 * np.random.uniform() * np.amax(data)  # 固定0.035
```

**影响**:
- 可能不适合所有类型的数据
- 无法自适应调整

**改进方向**: 实现自适应数据增强，根据样本特性调整参数。

### 2. 特征提取计算量大
**问题**: 94维特征提取需要较多计算
- MFCC: 40维
- Mel频谱: 40维
- Chroma: 12维
- 实时性受限

**影响**:
- 实时应用困难
- 大规模数据处理慢

**优化方案**: 特征缓存、并行计算、简化特征集。

### 3. 数据集依赖强
**问题**: 特征提取参数针对C-BESD数据集优化
- 采样率: 16kHz
- 音频长度: 2.5秒
- 儿童语音特性

**影响**:
- 迁移到其他数据集需要重新调整
- 成人语音可能效果不同

**处理方式**: 明确说明数据集依赖，提供适配指南。

## 实验设计问题

### 1. 评估指标不完整
**问题**: 主要关注准确率，其他指标记录不全
```
现有: 准确率、Macro F1、Recall
缺失: Precision per class、Confusion matrix、ROC-AUC
```

**影响**:
- 无法全面评估模型性能
- 类别不平衡分析不足

**解决方案**: 已建议在后续实验中补充完整评估指标。

### 2. 验证集使用不一致
**问题**: 部分实验可能使用了不同的验证集划分
- 早期实验划分方式可能不一致
- 未记录详细划分信息

**影响**:
- 结果可比性受影响
- 性能估计可能有偏差

**处理方式**: 在GitHub版中统一划分策略并详细记录。

### 3. 超参数搜索不足
**问题**: 超参数主要通过经验设置，未系统搜索
```
学习率: 5e-4 (经验值)
权重衰减: 5e-5 (经验值)
批量大小: 128 (硬件限制)
```

**影响**:
- 可能未达到最优性能
- 超参数选择缺乏依据

**改进方向**: 实现系统化的超参数搜索和记录。

## 实际应用限制

### 1. 实时性限制
**问题**: 完整流程延迟较高
```
特征提取: 50-100ms (CPU)
模型推理: 10-15ms (GPU)
总延迟: 60-115ms
```

**影响**:
- 实时交互应用可能延迟明显
- 需要优化才能达到实时要求

**优化建议**: 特征提取优化、模型量化、推理加速。

### 2. 环境依赖强
**问题**: 需要特定软件环境
```
必须: PyTorch、CUDA、特定库版本
建议: GPU、足够内存
```

**影响**:
- 部署复杂度高
- 资源受限环境无法运行

**轻量方案**: 提供CPU推理版本、模型简化选项。

### 3. 泛化能力未充分验证
**问题**: 主要在C-BESD数据集上测试
- 未在其他儿童语音数据集验证
- 未在成人语音数据集测试
- 跨语言泛化未知

**影响**:
- 实际应用效果不确定
- 需要额外验证

**验证建议**: 在更多数据集上进行交叉验证。

## 用户注意事项

### 运行前必须修改
1. 修改`src/data/dataset.py`中的硬编码路径
2. 添加随机种子设置代码
3. 准备数据集或使用模拟数据

### 运行期望管理
1. **不要期望直接运行**: 需要适配修改
2. **不要期望完全相同结果**: 存在随机性
3. **不要期望生产级代码**: 这是研究项目

### 合理使用预期
1. **适合**: 学习参考、二次开发、研究比较
2. **不适合**: 直接生产部署、商业应用、医疗诊断
