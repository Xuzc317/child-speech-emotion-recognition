## 第 3 页 | 原始方案概述

**标题**：原始论文方案（2025年5-7月）

**左栏 — 方案设计**：
- 任务：BESD 儿童双语数据集，6 类情绪分类
- 特征：162 维手工声学特征（ZCR+Chroma+MFCC+RMS+Mel）→ mean pooling
- 增强：每 WAV → 3x 处理（原始 + 加噪 + 拉伸变调）
- 模型：DrseCNN（ResNet 残差 + SE 通道注意力 + 多阶段卷积）
- 参数量：~11.37M（论文报告值）

**右栏 — 论文实验结果**：
| 模型 | Accuracy |
|------|----------|
| DrseCNN | **86.82%** |
| CNN-BiLSTM | 72.26% |
| CNN | 72.14% |
| Transformer | 69.48% |

> 要点：看起来效果很好，消融实验完整，对比模型齐全

## 第 4 页 | 转折一：一场面试，一次质疑

**标题**：CTO 的一个问题，让我重新审视整个实验

**故事背景**：
- 在一次求职面试中，我向面试官（公司 CTO）介绍这个项目
- 讲到"动态数据增强策略"时，CTO 打断了我的介绍

**CTO 的质疑**：
> "你先对每条语音做了加噪、拉伸、变调，生成了三个增强版本，然后再对整个数据集做随机划分。你有没有想过，同一条原始语音的增强版本，可能同时出现在训练集和测试集里？"

**我回去核查代码后发现的真相**：

```
原始流程：WAV → 3x增强 → random.shuffle → 70/30 切分
                               ↑
              同一 WAV 的原始版、加噪版、变调版
              有 63% 概率被分到训练集和测试集两边
```

**量化影响**：
- 12,537 条样本中，约 63% 存在 speaker 重叠
- 模型学的是"这句话的三个变体听起来像"，而非"这句话表达什么情绪"

> 要点：不是代码写错了——`random.shuffle` 本身没问题——而是增强和划分的先后顺序从根本上就是错的

---

## 第 5 页 | 顺着这个问题，挖出了更多隐患

**标题**：数据泄露只是冰山一角

| # | 问题 | 严重程度 |
|---|------|:---:|
| 1 | 数据泄露（speaker 重叠 63%） | 🔴 |
| 2 | 时间维度完全丢失 — mean pooling 抹平所有帧 | 🔴 |
| 3 | 增强顺序错误 — 测试集混入增强数据 | 🟠 |
| 4 | 仅用 ENGLISH 子集，非完整 MY 数据集 | 🟠 |
| 5-9 | 特征维度标注错、网络硬编码、论文数据不一致等 | 🟡 |

**最严重但最隐蔽的问题 — 时间维度丢失**：
```
WAV → 帧级特征 (T帧 × 162) → np.mean(所有帧) → (162,) 向量
                                  ↑ 时间信息在这一步完全消失

于是：DrseCNN Conv1d → kernel 在 ZCR/Chroma/MFCC 的特征拼接维度上滑动
     Transformer → (B, 1, 162) → T=1 → self-attention 恒等映射
     CRNN LSTM → 把下采样后的特征组边界当"时间步"
```

**所有"时序模型"都在一个没有时间维度的数据上运行。** 这是结构性错误，不是一个 bug。

## 第 6 页 | 系统性修复

**标题**：数据泄露修复（2026年4月25-26日）

**修复内容**：

| 修复项 | 操作 |
|--------|------|
| 说话人划分 | 从文件名提取 speaker ID → `normalize_speaker_id()` 修正变体 |
| 分层划分 | Profile-based stratification（按情感组合分组后 70/30） |
| 数据集 | ENGLISH → MY（ENGLISH+TELUGU，4,179 WAV，237 说话人） |
| 增强顺序 | 先划分 → 再增强（训练集和测试集各自独立处理） |
| 旧权重 | 全部移至 `legacy/`，标注无效 |

**验证**：
- 训练集 165 说话人 / 测试集 72 说话人，**0 重叠**
- 各类别分布均衡，每类训练 ~1,400-1,470 条，测试 ~630-645 条

## 第 7 页 | 修复后的真实结果

**标题**：数据泄露修复后 —— 真实基线

**实验结果**（同一 162-dim mean-pooled，仅修复 speaker 划分）：

| 模型 | 修复前 | 修复后 | 变化 |
|------|:---:|:---:|:---:|
| DrseCNN | 85.99% | **~35%** | -51pp |
| CNN | 72.14% | ~30% | -42pp |
| BiLSTM | 72.26% | ~30% | -42pp |
| Transformer | 69.48% | ~28% | -41pp |

**三个关键发现**：
1. 所有模型一起崩到 30-40%（随机基线 16.7%）
2. 模型间"性能梯度"消失 —— 原来的排序是泄露程度的排序
3. **162-dim mean-pooled 的信息上限 ≈ 35%**

**根本原因**（配图示意）：
```
所有"时序模型"都在没有时间维度的数据上运行
Conv1d 在特征拼接维度滑动 ≠ 时间建模
```

> 要点：不是模型不够好，是特征从根本上错了。这是整个项目最重要的转折点。

## 第 8 页 | 转折二：又一次面试，又一次提点

**标题**：在我差点放弃自监督学习的时候，另一位 CTO 给了新方向

**故事背景**：
- 入职现在这家公司时，跟 CTO 聊起项目经历
- 我说到：修复泄露后准确率只有 35%，试过 wav2vec2 做自监督特征提取但效果不好，所以觉得自监督学习跟这个项目不太匹配

**CTO 的回应**：
> "wav2vec2 效果不好，不代表自监督学习这条路不对。可能是你选的模型不合适——wav2vec2 是 2020 年的模型，预训练数据是成人朗读。也可能是你没调对。现在有更新的模型，比如 WavLM 在语音任务上普遍比 wav2vec2 强，emotion2vec 更是专门为情绪识别设计的。"

**他从原理层面给我梳理了关键点**：
- 自监督预训练模型学到的是**帧级别的连续表征**，天然保留了时间结构
- 我之前 162 维 mean-pooled 的问题恰恰是**丢掉了时间维度**
- 不是自监督不适合，而是需要选对模型 + 根据儿童语音特点做适配

**这次谈话让我重新审视了方向**：
- 之前放弃 SSL 是因为 wav2vec2 没跑通 + 自己调参能力不够
- 现在意识到：SSL 帧级特征 + 保留时间维，恰好是对症下药
- 但直接拿成人预训练模型用还不够 —— 儿童语音有独特的分布特征
- 由此形成了"分布驱动三模块"的整体思路

> 要点：这两次面试，第一次帮我发现了问题，第二次帮我找到了方向

## 第 10 页 | 模块 1 — 分布校准适配器

**标题**：模块 1 — AcousticCalibrationAdapter

**问题**：WavLM 在成人数据预训练，儿童语音存在分布偏移
- 定量证据：Frechet Distance = 0.37, scale 平均偏离 8%

**方法**：Frozen backbone + 轻量可学习非线性适配器

**消融结果**：

| 实验 | 配置 | Val Acc | vs 基线 |
|------|------|---------|---------|
| A1 | 无 Adapter | 80.66% | — |
| A2 | 统计先验初始化 | 81.45% | +0.79pp |
| **A2b** | **随机初始化** | **85.40%** | **+4.74pp** |

**反直觉发现** ⚡：
统计先验不如随机初始化 → 证明简单线性补偿不够，需要 Adapter 结构自己学习校准

---

## 第 11 页 | 模块 2 — 时序重要性池化

**标题**：模块 2 — TemporalImportancePooling

**问题**：儿童 F0 波动极大（p5=75Hz, p95=2286Hz），帧级情感密度不均匀

**方法**：用 F0 + energy 韵律特征引导帧级 attention

```
SSL 特征 (B, T, 768) ─→ 学习每帧权重
韵律特征 (B, T, 2) ──→ 调节注意力       → 加权求和 → (B, 768)
```

**结果**：
- Adapter + Prosody Pooling → 86.42%（+1.02pp over Adapter-only）
- 韵律引导 > 纯 self-attention → 有可解释性优势

---

## 第 12 页 | 模块 3 — 分布约束增强（负面实验）

**标题**：模块 3 — Distribution-Constrained Augmentation

**动机**：成人增强参数（±6st, 0.7-1.3）应用到儿童可能破坏情感特征

**儿童语音统计**：F0 mean=495.9Hz, std=651.1Hz → 约束参数：±3st, 0.85-1.15

**四组对比实验**：

| 实验 | FD | Val Acc | vs C1 |
|------|-----|---------|-------|
| C1 (无增强) | 0 | **86.42%** | 基线 |
| C3 (儿童约束) | 8.71 | 66.85% | **-19.57pp** |
| C2 (成人参数) | 9.87 | 59.51% | -26.91pp |
| C4 (极端参数) | 11.99 | 43.25% | -43.17pp |

**[配图] FD vs Accuracy 严格单调负相关曲线**

**核心发现**：分布偏移可定量预测分类退化

> 要点：这张图是论文最有说服力的证据之一
