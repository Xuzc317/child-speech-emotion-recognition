## 一、本次为什么修改

原 v4 方案存在以下问题：

1. **数据划分协议**：使用 60/20/20 三路直接切分，不够传统。改为"外部 7:3 + 内部 val"更符合 SER 领域的论文惯例，本质仍是三集合 train/val/test，但表述更自然。

2. **模块叙事**：原先将 Adapter、Prosody Pooling、Augmentation 并列为"三个技术模块"，但增强实验显示**无增强效果最好**。将"提升性能的增强方法"写成方法模块不诚实。

3. **关键实验问题**：
   - A3 标注为 "Full Fine-tune"，但 precomputed 模式下 `use_backbone=False`，WavLM 从未进入训练图，**不是真正的 full fine-tune**。
   - B2 标注为 "Self-Attention Pooling"，但 `use_prosody=False` 时回退到 mean pooling，**与 A2b 完全相同**。

4. **隐性技术问题**：padding mask 在 collate 中生成了但未被模型使用；数据集数字和 speaker 数量在不同文档中不一致。

## 二、最终采用的数据划分协议

### 协议名称：外部 7:3 + 内部 val

```
Step 1: 外部 speaker-independent 7:3 划分
  - 70% speaker → trainval
  - 30% speaker → test (严格 hold-out)

Step 2: 内部从 trainval 划分 val
  - trainval 内部 85% train / 15% val
  - 最终约: train 59.5%, val 10.5%, test 30%
```

### 约束规则

1. train/val/test speaker 三者互斥（profile-stratified，多层断言）
2. adapter_init 只用 train speaker（不用 val，不用 test）
3. val 仅用于 early stopping 和模型选择
4. test 严格 hold-out，训练中完全不可见
5. 禁止 test set = val set
6. 禁止用 test speaker 计算任何统计先验

### 实现

- 新函数：`src/data/preprocess.py:split_speakers_7_3_with_inner_val()`
- 旧函数保留：`split_speakers()` (70/30), `split_speakers_3way()` (60/20/20) 标记为 DEPRECATED
- 所有特征提取脚本已更新使用新函数
- `train_ssl.py` val 缺失时**报错退出**（不再 fallback 到 test）

## 三、模块 1/2/3 新叙事

### 最终方法模块（两个）

**模块 1 — Acoustic Calibration Adapter**

一句话：解决"每一帧的特征对不对"。

WavLM 是成人语音预训练模型 → 儿童语音进入后帧级特征存在域偏移（FD=0.37） → Adapter 对每一帧 768 维特征做轻量校准（scale + bias + gate） → 输出维度不变，不改变时间长度。

关键实验：A2b vs A1: +1.17pp (Phase 3 strict test)

**模块 2 — Prosody-guided Temporal Importance Pooling**

一句话：解决"哪些帧更重要"。

WavLM 输出 200 帧特征，简单 mean pooling 一视同仁 → 儿童语音情绪信息集中在音高/能量变化明显的帧 → 用 F0 和 energy 引导 attention 加权 → 输出 768 维句子向量。

关键实验：B3 vs A2b: +1.73pp (Phase 3 strict test)

**两者关系**：WavLM → Adapter (校准每帧) → Prosody Pooling (加权重要帧) → DrseNet (分类)

### 模块 3：增强敏感性分析（实验章节）

**不是方法模块**，放在实验章节。

标题："Distribution Shift Sensitivity Analysis" 或"增强敏感性分析：分布偏移导致分类退化"

核心论证：
- 实验目的不是证明增强提升性能
- 而是证明成人 SER 中常用增强不能直接迁移到儿童 SER
- FD 越大，准确率越低（严格单调）→ 分布偏移可定量解释性能退化
- 最终推荐模型**不使用波形增强**

## 四、A3/B2/模块 3 的处理决定

| 问题 | 决定 | 原因 |
|------|------|------|
| A3 Full Fine-tune | 降级/删除，不进入论文主消融表 | use_backbone=False → WavLM 不在训练图，不是真 fine-tune |
| B2 Self-Attention | 删除对比 | use_prosody=False → 回退到 mean pooling，=A2b |
| 模块 3 增强 | 移至实验章节，改称"增强敏感性分析" | 无增强效果最好，不能作为"性能提升模块" |

### 最终推荐模型

**B3**: WavLM frozen + Adapter (random init) + Prosody Pooling + DrseNet

Phase 3 严格 test 结果：**81.24% ± 1.45%**（历史 60/20/20 协议）

模块增益（Phase 3 严格 test）：
- Adapter: +1.17pp (A1 78.34% → A2b 79.51%)
- Prosody Pooling: +1.73pp (A2b 79.51% → B3 81.24%)
