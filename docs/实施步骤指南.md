# 实施步骤指南

> 版本：v1 | 日期：2026-04-27
> 目标：从当前 30-40% 的基线推进到可发表的儿童 SER 框架

---

## 整体路线图

```
Phase 1 ── 验证 SSL 基线 ──→ Phase 2 ── 逐个实现三模块 ──→ Phase 3 ── 整合实验 ──→ Phase 4 ── 论文撰写
  (1-2 天)                     (1-2 周)                     (1 周)                     (2 周)
```

每个阶段有明确的**进入条件**和**验收条件**，满足后再进入下一阶段。

## Phase 1：验证 SSL 基线（1-2 天）

### 目标
确认 emotion2vec/WavLM 在 BESD MY 上能解决"162-dim mean-pooled 只有 30-40%"的问题。

### 进入条件
- [ ] BESD MY 数据集路径可用
- [ ] CUDA 环境可用（RTX 3060 16GB 够用）

### 步骤

#### Step 1.1：安装依赖

```bash
# 进入新项目目录
cd D:/大学/论文/儿童语音情绪识别/新方案-分布驱动儿童SER

# 基础依赖（旧项目已有）
pip install torch torchaudio librosa numpy scipy tqdm matplotlib

# 新增：HuggingFace transformers（用于加载 SSL 模型）
pip install transformers
# 或 fairseq（如果 emotion2vec 需要）
# pip install fairseq
```

#### Step 1.2：确认 emotion2vec 模型可加载

```python
# 验证脚本
from transformers import Wav2Vec2Model
import torch

# 尝试加载 emotion2vec
# 注意：需要先确认 HuggingFace 上的正确 model_id
# 可能是以下之一：
#   - "emotion2vec/emotion2vec_base"
#   - "ddw1997/emotion2vec_base"
#   - 或从 GitHub 仓库下载
model = Wav2Vec2Model.from_pretrained("emotion2vec/emotion2vec_base")
print(f"Loaded: {model.config.hidden_size} dim")  # 预期 768

# 如果 emotion2vec 加载失败，fallback 到 WavLM
# model = WavLMForAudioClassification.from_pretrained("microsoft/wavlm-base")
```

**如果 emotion2vec 不可用，立即切到 WavLM Base**，不要卡在这一步。WavLM 已有 huggingface 官方权重，且 `jihedjabnoun/wavlm-base-emotion` 已经在 SER 上有发布。

#### Step 1.3：预提取 SSL 特征

编写 `scripts/extract_ssl_features.py`：

```bash
python scripts/extract_ssl_features.py \
    --wav_dir /path/to/BESD/MY \
    --output_dir data/ \
    --model emotion2vec \
    --device cuda
```

这个脚本做的事：
1. 加载 frozen SSL backbone
2. 遍历 BESD MY 的所有 WAV（与 preprocess.py 的 collect_wav_files 逻辑一致）
3. 对每个 WAV，跑一次 backbone → 输出 (T, 768) 帧级特征
4. 使用 speaker-independent 划分（preprocess.py 已有）→ 保存为 train/test 两个 .npy

**注意特征对齐问题**：
- SSL 模型的帧移通常是 20ms（50Hz），2.5 秒音频 ≈ 125 帧
- 必要时需要做时间对齐处理

#### Step 1.4：跑 linear probe（Phase 1 关键实验）

```bash
python src/training/train_ssl.py \
    --mode precomputed \
    --train_feats data/train_ssl_feats.npy \
    --train_labels data/train_labels.npy \
    --test_feats data/test_ssl_feats.npy \
    --test_labels data/test_labels.npy \
    --lr 3e-4 \
    --batch_size 64 \
    --epochs 100
```

当前配置：backbone frozen → adapter disabled → prosody disabled → 仅 DrseCNN 分类器。

**预期结果与决策**：

| Linear Probe 准确率 | 结论 | 下一步 |
|-------------------|------|--------|
| > 60% | SSL 特征有效，时间维是关键 | 进入 Phase 2 |
| 45-60% | 有效但不够，需要 adapter | 进入 Phase 2，优先做 adapter |
| 30-45% | 跟之前差不多，不太对 | 排查问题（帧对齐？特征维度？模型加载？） |
| < 30% | 有问题或 SSL 模型不适合 | 换 WavLM 试，或重新评估方向 |

### 验收条件
- [ ] SSL linear probe 准确率显著 > 40%
- [ ] 确认保留时间维度的价值
- [ ] 如果成功，进入 Phase 2

## Phase 2：逐个实现三个模块（1-2 周）

### Sub-Phase 2.1：模块 1 — 分布校准适配器（3-5 天）

#### 目标
在 frozen backbone 上加入 Adapter，验证校准是否优于 linear probe。

#### Step 2.1.1：统计先验初始化

```bash
# 1. 在 BESD MY 上跑一次 frozen backbone，收集帧级特征
# 2. 计算 per-dimension mean/std
# 3. 与成人数据集（如 IEMOCAP 或 RAVDESS）的特征统计做差
# 4. 用这个差值初始化 Adapter 的 scale/bias
```

在 `scripts/` 下写 `compute_adapter_init.py`，输入是 SSL 特征统计，输出是适配器初始化参数。

#### Step 2.1.2：实现 Ablation 对比

| 实验 | Adapter | Backbone | 预期效果 |
|------|---------|----------|---------|
| A1 | 无（linear probe） | frozen | 基线 |
| A2 | AcousticCalibration | frozen | 提升 > A1 |
| A3 | 无 | full fine-tune | 可能最高但贵 |
| A4 | DomainAdversarial | frozen | 如果有成人数据 |

#### Step 2.1.3：验证

```bash
python src/training/train_ssl.py --use_adapter --frozen_backbone
```

关键检查点：Adapter 的加入是否带来了稳定的准确率提升。

#### 验收条件
- [ ] AcousticCalibrationAdapter 实现并跑通
- [ ] Ablation 显示 adapter 有提升
- [ ] adapter 初始化方式有消融对比（统计先验 vs 随机初始化）

### Sub-Phase 2.2：模块 2 — 时序重要性池化（3-5 天）

#### 目标
用韵律特征（F0 + energy）引导帧级 attention，替代简单 mean pooling。

#### Step 2.2.1：韵律特征提取工具

```python
# 验证帧级 F0/energy 提取是否正确
# 使用 src/models/pooling.py 中的 extract_prosody()
# 可视化对比不同情感类别的 F0 轮廓
# 预期：ANGER 的 F0 变异性 > NEUTRAL 的 F0 平坦度
python scripts/analyze_prosody.py
```

这个步骤本身可以出图，放在论文的 "Statistical Analysis" 章节。

#### Step 2.2.2：实现 TemporalImportancePooling

```python
# 框架：
#   1. 从 SSL feature 模型中获取帧对应的时间位置
#   2. 用 librosa 对齐提取 F0 + energy
#   3. 拼接后过 attention → 得到每帧权重
#   4. 加权求和
```

#### Step 2.2.3：Ablation 对比

| 实验 | Pooling | 预期效果 |
|------|---------|---------|
| B1 | mean pooling | 基线 |
| B2 | self-attention pooling（无韵律） | 提升 |
| B3 | TemporalImportancePooling（韵律+特征） | 提升 > B3 |
| B4 | 只用韵律权重（无 SSL 特征 attention） | 对比可解释性 |

通过 B2 vs B3 的对比，证明韵律引导的 attention 优于纯数据驱动的 attention。

#### 验收条件
- [ ] 韵律特征提取在 BESD MY 上跑通
- [ ] TemporalImportancePooling 实现并跑通
- [ ] Ablation 显示韵律引导比纯 attention 更好

### Sub-Phase 2.3：模块 3 — 分布约束增强 + 负面实验（3-5 天）

#### Step 2.3.1：计算儿童语音统计分布

# 用 src/augmentation/constrained_aug.py 的 compute_child_speech_stats()

#### Step 2.3.2：切到在线训练模式

## 关键风险点

1. **emotion2vec 加载不成功** → 立即 fallback 到 WavLM Base，不要卡
2. **预提取特征时间太长**（4,179 WAVs × 每条几秒）→ 先跑 500 条验证曲线，确认后再全量
3. **Phase 2 online 模式太慢** → 可以考虑用预提取特征 + 在 (T, 768) 层面做 SpecAugment，替代波形级增强
4. **跨语言迁移效果太差**（Telugu 完全学不到）→ 至少保留负面结果，作为"分布偏移严重"的极端案例
