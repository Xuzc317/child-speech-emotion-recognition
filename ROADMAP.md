## 优先级 1：必须修复

### 说话人无关划分（数据泄露） ✅ 已修复 (2026-04-25)

**问题**：论文声称按说话人（70%/30%）划分训练/测试集，但所有代码实际使用的是对整个数据集做随机打乱划分（`random.shuffle` / `random_split`）。同一说话人的不同增强版本会同时出现在训练集和测试集中，约63%的原始语音存在此数据泄露问题。

**修复内容**：
1. 从文件名提取说话人ID（如 `1.EF_12 Angry_1.wav` → `1.EF_12`）
2. 数据集从 `ENGLISH/` 切换至 `MY/`（ENGLISH + TELUGU，共 4,179 WAV，237 名唯一说话人）
3. 按说话人情绪档案分层划分（profile-based stratification）：说话人按其拥有的情感类别组合分组，每组内 70/30 划分，确保各类别在训练/测试间分布均衡
4. 训练集和测试集均做 3x 数据处理（原始 + 加噪 + 拉伸变调）
5. 生成文件：`train_data.npy` / `train_labels.npy` / `test_data.npy` / `test_labels.npy`
6. 说话人 ID 标准化：修正连字符（`1.EF-12` → `1.EF_12`）和缺失点号（`3EF_9` → `3.EF_9`）
7. 需重新训练模型以获得准确的说话人无关评估结果

### 数据增强顺序修正 ✅ 已修复 (2026-04-25)
