# 儿童语音情感识别 (SER) 顶会项目：全模块重构与备选方案矩阵

---

## 模块 1: 数据基建与跨库评估流水线 (Data Pipeline)

**当前状态：** 仅使用 C-BESD 小数据集；已修复数据泄露，但包含有害的音频约束增强（时间拉伸/音高平移）。

**👑 最优解决方案：多库融合与严格标准化流水线**

- **做法：** 引入 FAU Aibo、MyST、KidsTALC 等标准儿童数据集。强制执行 16kHz/Mono 重采样；将标签统一映射为（怒、喜、悲、中性）四分类；基于 Hash ID 实现绝对的说话人隔离（Speaker-disjoint）；彻底废弃音高等破坏性增强，仅保留加性环境噪声（如 MUSAN 库）。
- **优势：** 直接突破单一数据集瓶颈，建立顶会极为看重的 Zero-shot/Cross-corpus 评估标准。

**备选方案 A：引入成人数据进行混合预热 (Mixed Pre-training)**

- **做法：** 先在 IEMOCAP 等成人数据集上训练，再在 C-BESD 上做 Few-shot 微调。
- **优劣：** 能缓解数据量不足（优）；但可能会将成人语音的声学流形"强加"给儿童数据，削弱模型对儿童特异性特征的捕捉（劣）。

**备选方案 B：保留单库 + 极度严苛的 K-Fold 交叉验证**

- **做法：** 仅使用 C-BESD，但实现 10-Fold 说话人独立的交叉验证。
- **优劣：** 工程量最小（优）；但在反驳审稿人关于"泛化性不足"的质疑时显得单薄（劣）。

## 模块 2: 特征基座与层级分析 (WavLM Backbone)

**当前状态：** 仅提取冻结的 WavLM Base+ 最后一层（第 12 层）输出特征。

**👑 最优解决方案：可学习权重加权和 (Learnable Weighted Sum)**

- **做法：** 提取全部 12 层隐藏层特征，初始化 12 个可学习参数，经 Softmax 后加权求和。
- **优势：** 解释性极强。训练后可直接输出 12 个权重值，绘制图表证明儿童数据更依赖浅层（声学层），而成人数据依赖深层（语义层），完美契合我们的"声学先验"故事线。

**备选方案 A：Gumbel-Softmax 层选择 (Gumbel Layer Selection)**

- **做法：** 使用 Gumbel-Softmax 迫使模型在训练中进行离散的"硬选择"，最终只输出某一层的特征。
- **优劣：** 推理时可直接抛弃不需要的计算层，极大提升端侧部署的速度（优）；但训练时温度参数（Temperature）难以调参，且不如加权和的分布图直观（劣）。

**备选方案 B：截断式前向传播 (Truncated Forward)**

- **做法：** 基于经验直接丢弃 WavLM 的后 4 层，仅使用前 8 层提取声学特征。
- **优劣：** 显存占用直降 30%（优）；但缺乏让模型自适应寻找最优层的科学论证过程（劣）。

---

## 模块 3: 时序池化机制 (Attention Pooling) — 【核心创新区】

**当前状态：** 已实现 F0 + RMS 引导的 Temporal Importance Pooling，取代了 Mean Pooling。

**👑 最优解决方案：引入参数对等的纯自注意力基线 (Symmetrical Self-Attention Baseline)**

- **做法：** 在代码中新增一个经典的纯数据驱动 Self-Attention Pooling 模块。确保其投影线性层（Linear Layer）的参数量与你的"韵律引导池化层"完全一致。通过 config 文件实现一键切换。
- **优势：** 这是防御审稿人"伪创新"攻击的最强武器，通过严格的控制变量法，证明性能增益纯粹来源于"F0+RMS 物理先验"，而非多出的模型参数。

**备选方案 A：多头 QKV 池化基线 (Multi-head QKV Pooling)**

- **做法：** 复现 2026 年 ICASSP 中常用于 Whisper 的多头注意力池化作为对比。
- **优劣：** 对比基线更加前沿（优）；但 QKV 结构过于复杂，容易引入额外的混淆变量，使得"消融实验"不够纯粹（劣）。

**备选方案 B：基于门控循环单元的时序池化 (GRU-based Pooling)**

- **做法：** 引入一个轻量级的单向 GRU 来处理时序聚合。
- **优劣：** 能捕获长时依赖（优）；但黑盒属性太重，破坏了注意力机制带来的帧级别可解释性（劣）。

## 模块 4: 下游分类器 (Classifier)

**当前状态：** 采用 SEMLP（带 Squeeze-and-Excitation 门控的多层感知机：768→512→256→128→6）。

**👑 最优解决方案：保持不变 (No Change)**

- **做法：** 直接沿用 SEMLP。
- **优势：** 在底层特征（模块2）和池化机制（模块3）同时发生剧烈变动时，保持顶层分类器不变，能够最大程度确保实验跑出的指标变化（增益或下降）完全归因于我们的核心创新点，避免"一调全调"导致的因果关系混乱。

**备选方案 A：替换为简单的线性映射 (Vanilla Linear Layer)**

- **做法：** 将池化后的特征直接通过一个 Linear(768, num_classes) 映射输出。
- **优劣：** 最严苛的 Linear Probing 测试，完全逼出 WavLM 表征的潜力（优）；但可能导致整体准确率绝对值下降，不够"好看"（劣）。

**备选方案 B：引入 1D-CNN 分类器**

- **做法：** 在全连接层前增加 1D 卷积。
- **优劣：** 能捕获局部特征（优）；但与池化层的功能存在重叠（劣）。

## 模块 5: 物理机理解释与可视化 (XAI Visualization)

**当前状态：** 无相关代码，模型处于黑盒状态。

**👑 最优解决方案：一维声学注意力热力图 + 相关性统计 (1D Saliency Maps & APC)**

- **做法：** 编写独立的可视化脚本。提取测试样本的时间轴，将（1）原始波形包络、（2）F0/RMS 的归一化曲线、（3）池化层的帧级权重 $\mathbf{A}_{weight}$，绘制在三联对齐图上。同时计算权重与 F0 导数的皮尔逊相关系数（Attention-Prosody Correlation, APC）。
- **优势：** 计算开销极小，直击视觉痛点。用最直白的证据告诉 Area Chair："看，我的模型确实在高亮标注孩子尖叫和情绪爆发的那一瞬间"。

**备选方案 A：积分梯度归因分析 (Integrated Gradients / SHAP)**

- **做法：** 利用 Captum 或 SHAP 库，计算输入音频信号对最终分类输出的梯度贡献值。
- **优劣：** 算法更高级、更普适（优）；但对于 768 维的高维特征难以映射回具体的"声学物理意义"，容易让审稿人看不懂（劣）。

**备选方案 B：引入 LLM 做思维链解释 (CoT with SpeechLLM)**

- **做法：** 接入大模型输出文字解释。
- **优劣：** 极其前沿，自带流量（优）；但工程量庞大，偏离了当前侧重"声学特征空间分析"的核心路线，更适合作为未来的子课题（劣）。

## 模块 6: 分布偏移诊断探针 (Distribution Shift Analytics)

**当前状态：** 已编写 Frechet Distance (FD) 代码，用于量化年龄、增强策略引发的偏移。

**👑 最优解决方案：FD + SMMD 双重指标校验库**

- **做法：** 在现有 FD 代码的基础上，让 Claude Code 补充基于高斯核函数的 Speech Maximum Mean Discrepancy (SMMD) 计算模块。在进行偏移量化时，同时输出 FD 和 SMMD 的值。
- **优势：** FD 强制假设语音特征服从高斯分布，这是顶会审稿人（特别是擅长统计学的 Reviewer）极易攻击的软肋。引入无分布假设的 SMMD，能在数学严谨性上打造"双保险"。

**备选方案 A：追加 Mardia 偏度测试模块**

- **做法：** 仅计算 FD，但补充一个统计学测试模块，验证 768 维特征是否真的满足高斯分布。
- **优劣：** 严谨（优）；但万一测试结果是不满足（语音特征常有此问题），反而会削弱 FD 存在的合法性（劣）。

**备选方案 B：降维可视化探针 (t-SNE / UMAP Visualization)**

- **做法：** 放弃复杂的拓扑距离计算，直接将特征降至 2D 并在二维图上画出成人和儿童特征簇。
- **优劣：** 直观好看（优）；但无法形成精确的"偏移量 vs 准确率"的数学单调性定律，缺乏理论深度（劣）。
